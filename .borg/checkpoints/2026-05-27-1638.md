## 1. Goal

Implement Directive B — migrate volatile session state out of the shared registry into per-project
`.borg/state.json` files so the registry becomes a pure discovery index.

## 2. Accomplished

**PR #34 merged — feat(state): Directive B — per-project state.json migration (v0.8.0)**

- **`lib/borg-hooks.sh`** — Added `_borg_resolve_proj_dir`, `_borg_state_file`, `_borg_state_read`,
  `_borg_state_write` bash helpers. Resolved code smell from /simplify: extracted repeated PROJ_DIR
  resolution block (3 identical copies across hooks) into `_borg_resolve_proj_dir`; replaced
  `$(dirname ...)` subprocess with `${sf%/*}`.

- **`lib/registry.zsh`** — Added `borg_state_{file,read,write}`, `borg_registry_with_state()`,
  `borg_registry_get_with_state()`. Removed volatile fields (`claude_session_id`, `last_activity`,
  `status`) from `borg_registry_add` schema. Updated `borg_registry_set_status` to write to
  state.json; removed dead `sf` variable.

- **`hooks/borg-link-down.sh`** — State update writes to state.json (not registry). Capacity
  warning scans state.json files. Uncommitted-changes reminder reads from state.json. Orchestrator
  overview reads status/last_activity from per-project state.json.

- **`hooks/borg-link-up.sh`** — Writes `status=idle` + `has_uncommitted_changes` +
  `claude_session_id` to state.json in one atomic write.

- **`hooks/borg-notify.sh`** — Writes `status=waiting` + `waiting_reason` + `notify_origin` to
  state.json. Fixed: now uses `_borg_find_project` (not `basename`) for marker resolution.

- **`borg.zsh`** — Bumped `BORG_VERSION` to v0.8.0. Updated `_borg_active_count`,
  `_borg_link_porcelain`, `_borg_link_overview`, `cmd_ls` to use `borg_registry_with_state`.
  Updated `_borg_link_deep`, `cmd_status`, `_borg_do_switch` to use `borg_registry_get_with_state`.
  Fixed gitignore bug in `cmd_setup` (was adding `.borg/` — breaking `!.borg/checkpoints/` — now
  adds `.borg/state.json`). Added state.json initialisation per project on setup.

- **`tests/lifecycle.bats`** — Updated all volatile-field assertions to read from state.json.
  Updated registry seeds to use real `TEST_CWD` paths. Updated orchestrator-mode tests to verify
  state.json NOT created.

- **`tests/registry.bats`** — Updated `borg_registry_set_status` test for state.json.

- **`tests/state.bats`** (new, 14 tests) — Covers all bash and zsh state helpers,
  `borg_registry_with_state`, `borg_registry_get_with_state`.

- **197/197 tests pass.** Deployed live via `borg setup`. All hooks in sync.
