## 2. Accomplished

**PR #66 merged and installed** (`5cb965c`). Squash-merged, `./install.sh` run, launchd agent registered.

**Found the installed poller was silently blind — and fixed it (PR #67, merged, `63b232e`).**

The agent fired every 120 s, exited `0`, and wrote **zero** samples. `launchctl list` reported it healthy. This was
caught only by verifying the install end-to-end; the 325-test suite was green the whole time.

Two defects, one symptom:

1. **`claude` was unresolvable under launchd.** The native installer puts it at `$HOME/.local/bin`, which launchd omits
   from its minimal `PATH`. Every manual run from a login shell passed — a login shell has that dir on `PATH`. Only the
   launchd invocation failed. The plist's `PATH` was `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin`.
2. **A missing binary was reported as a parse failure.** `output=$(...) || output=""` collapsed a *permanent
   misconfiguration* into a *transient format drift*, logged `WARNING`, and exited `0`.

Defect 2 is why defect 1 survived install. Fix: prepend `$HOME/.local/bin` to `PATH` in the script (not the plist — the
script self-heals for every invocation path, so there is one source of truth), and add a `command -v "$CLAUDE_BIN"`
preflight that logs `ERROR` and **exits nonzero**, so a misconfigured poller surfaces as an unhealthy launchd job.

`/simplify` pass folded a duplicate test into the existing empty-output case and collapsed a two-grep assertion into one
same-line pattern.

Verified: **327/327 bats**, shellcheck clean, and the new missing-binary test **fails against the pre-fix script** (it
exited 0) — a real regression pin, not a tautology. Live launchd kickstart from `main` now writes samples.
