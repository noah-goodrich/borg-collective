## 1. Goal

## 2. Accomplished
- **Diagnosed ssh-agent root cause:** Docker Desktop's grpcfuse was stamping mode 600 back onto `~/.ssh/agent/` via
  `com.docker.grpcfuse.ownership` xattr, stripping the x-bit. ssh-agent's `agent_cleanup_stale()` then failed EACCES,
  exited 255, and launchd throttled respawns until the host agent stayed dead. SIP blocks `launchctl kickstart`, so
  recovery was non-trivial.
- **Immediate unblock:** killed wedged ssh-agent/ssh-add processes, `chmod 700 ~/.ssh/agent`, `xattr -d
  com.docker.grpcfuse.ownership ~/.ssh/agent`, loaded key via `ssh-add --apple-use-keychain ~/.ssh/id_ed25519`,
  verified `ssh -T git@github.com` works.
- **Permanent fix across 14 files:**
  - Templates (4): `borg-collective/dotfiles/devcontainer/{docker-compose.base.yml, Dockerfile.base}` +
    `~/.config/dotfiles/devcontainer/` equivalents
  - Scaffold templates (3): `claude-plugins/dev-tools/skills/bootstrap-project/references/docker-compose.template.yml`,
    `borg-collective/templates/supabase/docker-compose.yml`, `borg-collective/drone.zsh:1033` embedded scaffold
  - Per-project compose files (7): reveal, pytest-coverage-impact, ingle, cairn, snowflake-projects, snowfort,
    snowfort-scaffold-bak
  - New pattern: selective `~/.ssh/config:ro` + `~/.ssh/known_hosts:ro` mounts instead of full `~/.ssh` directory
    bind; base Dockerfile pre-creates `/home/dev/.ssh` (mode 700, dev-owned)
- **Self-heal:** added `heal_ssh_agent_dir()` to both `install.sh` copies — fixes perms and strips grpcfuse xattr on
  re-run.

  `~/.config/dotfiles/claude/plugins/token-cost/...` to live marketplace source
