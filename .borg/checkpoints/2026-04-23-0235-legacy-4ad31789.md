## Decisions Made

- **Host daemon over in-container approach:** `borg-notifyd` watches the shared state file on the host via `fswatch` rather than attempting `osascript` from inside the container — avoids the fundamental macOS security barrier.
- **Silent startup snapshot:** Daemon takes a baseline state snapshot on launch so it doesn't fire spurious popups for pre-existing `waiting` projects.
- **Three patch releases instead of one:** Each fix that broke work-machine setup (atomic write, missing `mkdir -p`, dangling symlink) got its own release to keep the Homebrew formula history clean and bisectable.
- **Supabase MCP: mutating ops stay behind prompts:** `execute_sql` and `apply_migration` were deliberately excluded from the allowlist — the risk of silent schema changes outweighs the prompt friction.

## Blockers and Observations

- **AC2 (no double-fire from host origin):** Cannot be verified without a live tmux session ending — requires a real Claude turn to complete. Root cause risk: `borg-notifyd` could fire and the in-container `notify.sh` could also fire if the `/.dockerenv` guard were missing; the guard is in place but untested end-to-end.
- **AC6 (click-to-focus):** Requires clicking the actual popup and observing Ghostty + tmux window switch — no way to automate.
- **Work machine not fully verified:** v0.7.5 resolved the dangling symlink issue on the work machine setup path, but a full end-to-end `borg setup` run hasn't been confirmed there.
- **`ingle` project invisible on host:** Runs entirely inside a drone container under `/workspace`, so `borg ls` shows it idle and no host-side session history exists — expected behavior, not a bug.
