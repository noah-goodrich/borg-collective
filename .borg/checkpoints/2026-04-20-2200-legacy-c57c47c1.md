## Decisions Made

- **`borg start` is a pure filesystem promotion, not a new plan format.** `docs/plans/directives/` = backlog; `PROJECT_PLAN.md` = in-flight; `docs/plans/assimilated/` = done. One in-flight plan per project enforced by file existence check. No schema changes needed ‚Äî `borg-assimilate` already handles the assimilated side.
- **git mv vs plain mv:** `cmd_start` checks `git ls-files` first; if the directive is tracked it uses `git mv` to preserve history, otherwise falls back to plain `mv`. Keeps the untracked-directive case (written mid-session) working.
- **Container-notification-bridge goes in `bin/borg-notifyd` + `launchd/` (not `borg init`)**: launchd agent pattern chosen over `borg init` background loop so the watcher survives shell restarts and is manageable independently.
